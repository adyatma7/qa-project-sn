# BUG-002: Fund Transfer does not enforce a sufficient-balance check

**Severity:** High
**Priority:** P1
**Environment:** Chromium via Playwright, ParaBank public demo, 2026-07-18

## Steps to Reproduce
1. Log in as `john` / `demo`
2. Go to `transfer.htm`
3. Enter amount: `999999999`
4. Leave "from" and "to" account as default selection
5. Click "Transfer"

## Expected Result
Transfer rejected — amount far exceeds any realistic account balance.

## Actual Result
"Transfer Complete!" is shown. The transfer is accepted regardless of the
account's actual balance.

## Evidence
Confirmed via `playwright-tests/tests/ui/transfer/test_transfer.py::test_transfer_exceeding_balance`,
using an assertion that explicitly waits 2 seconds for the page to settle
before checking (see DEC-012) — ruling out the race-condition false
negative that affected this exact test case's earlier "pass." Full page
snapshot at failure time confirms a genuine, unambiguous
`<h1 class="title">Transfer Complete!</h1>`, not a partial/loading state.

## Notes
- **Confidence: now cross-tool confirmed.** Reproduced 2/2 in Playwright
  (`xfail` as expected both times) and confirmed in Selenium too (same
  "Transfer Complete!" result on the amount that should be rejected).
  Upgraded from "single confirmed run" — this is as solid as BUG-001's
  evidence now, arguably more so since it wasn't contradicted on retry.
- **Security/business implication, stated for portfolio purposes:** in a
  real banking application, an unenforced balance check on transfers
  would allow an account to go arbitrarily negative — a serious business
  logic flaw, not just a UX nicety.
- **Regression test:** `test_transfer_exceeding_balance` keeps asserting
  the correct expected behavior (no completion) — not rewritten to match
  the bug — so it will fail until this is fixed, serving as the
  regression guard. Marked `xfail(strict=False)`, consistent with
  BUG-001's precedent (DEC-012 area) — non-strict since only one
  confirmed run exists so far, not yet enough data to call it
  deterministic.
